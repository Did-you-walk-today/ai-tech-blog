# JSONHOUSE.COM — 콘텐츠 스타일 가이드

박종훈 스타일 | 메르 스타일 | 합성 스타일 (jsonhouse DNA)

---

## 이 가이드의 목적

jsonhouse.com의 모든 포스트는 아래 3가지 스타일 중 하나를 기반으로 작성됩니다.  
각 스타일은 타겟 독자와 콘텐츠 유형에 따라 선택합니다.  
핵심은 단순 정보 전달이 아닌, 독자가 읽고 나서 **'세상이 어떻게 돌아가는지 알게 된 느낌'** 을 주는 것입니다.

| 스타일 | 기반 | 타겟 | 주요 카테고리 |
|--------|------|------|--------------|
| 박종훈 스타일 | 거시 맥락 + 흐름 분석 | 투자자, 직장인, 정책 관심층 | CAT7 Deep Dive |
| 메르 스타일 | 쉬운 언어 + 실용 연결 | 일반인, 입문자, 취준생 | CAT7 Weekly Digest |
| 합성 스타일 (jsonhouse DNA) | 맥락 + 데이터 + 실용 | 개발자, AI 파워유저 | CAT1~6 전체 |

## ① 박종훈 스타일

> **핵심 철학**  
> "흩어진 뉴스들을 하나의 거대한 흐름으로 꿰어낸다. 주가가 올랐다·내렸다가 아니라, 왜 그 일이 벌어졌는지에 집중한다."

### 어투 & 톤

- 경제학자이자 기자 — 데이터와 서사를 함께 가져간다
- 역사적 맥락, 지정학적 각도를 반드시 포함한다
- 뉴스 나열 금지 — 반드시 하나의 '거대한 흐름'으로 연결한다
- 독자가 읽고 나면 세상의 큰 지도가 그려지는 느낌을 줘야 한다
- 어렵지 않되 깊이가 있다. 전문 용어는 반드시 바로 뒤에 쉬운 설명 추가

### 글 구조 (5단계)

| 단계 | 이름 | 목표 | 분량 |
|------|------|------|------|
| ① | 훅 (Hook) | "지금 왜 이게 중요한가?" — 독자를 붙잡는 도입부 | 2~3문장 |
| ② | 원재료 소개 | 리포트/기관이 뭐라고 했나 — 팩트 전달, 출처 명확히 | 3~5문단 |
| ③ ★ | **이면 분석** | 진짜 의미는 이거야 — 역사적 맥락, 구조적 원인 (핵심!) | 4~6문단 |
| ④ | 큰 그림 연결 | 다른 이슈들과 어떻게 연결되나 — 최소 2개 트렌드 연결 | 2~3문단 |
| ⑤ | 실용적 결론 | 그래서 투자자/직장인에게 어떤 의미인가 | 2~3문단 |

### 금지 사항

- "전문가들은 말한다" — 누가? 어디서? 출처 없는 인용 금지
- 단순 뉴스 요약으로 끝나는 포스트 금지
- 이면 분석(③) 없는 포스트 자동 거부

### 제목 패턴

```
✓ "IMF가 경고한 2026 글로벌 부채 위기 — 한국은 안전한가?"
✓ "트럼프 관세 2.0이 한국 수출에 미치는 진짜 영향"
✓ "AI가 없앤 직업 vs 만든 직업 — 2026 데이터로 보는 현실"
```

## ② 메르 스타일

> **핵심 철학**  
> "방대한 분량의 데이터를 누구나 이해할 수 있는 쉬운 언어로 풀어서 설명한다. 그 정보가 결국 투자나 실물 경제에 어떤 영향을 미치는지 연결해 준다."

### 어투 & 톤

- 복잡한 글로벌 리포트를 '아, 그렇구나!' 하게 만드는 쉬운 언어
- 경제·역사·정치·산업 넘나들며 '사건의 이면'을 파헤친다
- 친근한 해설자 — 독자 옆에서 설명해주는 느낌
- 매일 읽어도 트렌드에 뒤처지지 않게 만드는 밀도
- 결론에서 반드시 '투자'나 '실물 경제'로 연결

### 글 구조 (5단계)

| 단계 | 이름 | 목표 | 분량 |
|------|------|------|------|
| ① | 쉬운 도입 | 이게 뭔 얘기인지 한 줄로 — 어려운 용어 없이 시작 | 1~2문장 |
| ② | 배경 설명 | 왜 이런 일이 벌어졌나 — 역사·지정학 배경을 쉽게 | 3~4문단 |
| ③ ★ | **데이터 해석** | 숫자가 실제로 뭘 말하는지 — 일반인도 이해하게 | 3~5문단 |
| ④ | 실물 연결 | 이게 내 월급·집값·취업에 어떤 영향? | 2~3문단 |
| ⑤ | 한 줄 결론 | 오늘 기억해야 할 것 딱 하나 | 1~2문장 |

### 금지 사항

- 전문 용어 나열 후 설명 없이 넘어가는 것 금지
- '그래서 나한테 뭔 의미인가' 없이 끝나는 포스트 금지
- 800단어 이하 포스트 금지 — 밀도 있는 설명 필수

### 제목 패턴

```
✓ "이번 주 꼭 알아야 할 글로벌 AI 뉴스 5선"
✓ "휴머노이드 로봇이 쏟아지는 이유 — 쉽게 설명하면 이렇습니다"
✓ "AI가 취업시장을 바꾸는 방식 — 데이터로 보는 진짜 현실"
```

## ③ 합성 스타일 — jsonhouse DNA

> **핵심 철학**  
> 박종훈의 '맥락과 흐름' + 메르의 '쉬운 언어와 실용 연결' + jsonhouse의 '구조화된 데이터'.  
> 읽고 나면 세상의 큰 지도가 그려지면서, 데이터까지 손에 쥐어지는 느낌.

### 어투 & 톤

- 거시적 흐름 분석 (박종훈) + 쉬운 언어 (메르) + 구조화 데이터 (jsonhouse)
- 영어 포스트: 전문적 분석가 톤 / 한국어 포스트: 친근한 해설자 톤
- 모든 주장은 데이터로 뒷받침 — JSON 블록 또는 비교 표 필수
- '왜'와 '이면'에 집중하되, 개발자도 바로 쓸 수 있는 실용 정보 포함
- TL;DR 필수 — Featured Snippet 타겟팅

### 글 구조 (5단계)

| 단계 | 이름 | 목표 | 분량 |
|------|------|------|------|
| ① | 훅 + TL;DR | 핵심 질문 + 3~5줄 요약 — 스캔하는 독자 잡기 | 2~3문장 + 불릿 |
| ② | 팩트 + 데이터 | 출처 명확한 데이터 — JSON 블록 또는 표 필수 | 3~5문단 |
| ③ ★ | **이면 분석** | 왜 이 일이 벌어졌나 — 맥락과 구조적 원인 | 4~6문단 |
| ④ | 큰 그림 + 연결 | 다른 트렌드와 연결 + AI/테크 생태계 영향 | 2~3문단 |
| ⑤ | 실용 결론 | 개발자/파워유저/직장인 관점 시사점 + FAQ | 2~3문단 |

### 필수 포함 요소

| 요소 | 설명 | 없으면 |
|------|------|--------|
| TL;DR 블록 | 3~5개 불릿으로 핵심 요약 | 자동 거부 |
| JSON 블록 or 비교 표 | 구조화 데이터 최소 1개 | 자동 거부 |
| FAQ 섹션 | 최소 3개 질문 (People Also Ask 타겟) | 자동 거부 |
| 이면 분석 | 단순 요약 아닌 '왜' 분석 | 자동 거부 |
| data_updated 날짜 | 데이터 최신성 명시 | 자동 거부 |

### 제목 패턴

```
✓ "Claude vs GPT-5: 2026 Complete Benchmark — Data You Can Actually Use"
✓ "LLM API Pricing 2026: Full Comparison Table (Updated Weekly)"
✓ "AI Job Market Impact 2026: What the Data Actually Shows"
```

## ④ 스타일 선택 가이드

| 상황 | 선택 스타일 | 이유 |
|------|------------|------|
| CAT7 Weekly Digest (주간 뉴스) | 메르 스타일 | 일반 독자, 빠른 소화 |
| CAT7 Deep Dive (경제·사회 분석) | 박종훈 스타일 | 깊이 있는 맥락 분석 |
| CAT1~2 (LLM/개발툴 비교) | 합성 스타일 | 데이터 + 분석 동시 필요 |
| CAT3~4 (프롬프트/워크플로우) | 합성 스타일 (실용 강화) | 바로 쓸 수 있어야 함 |
| CAT5 (데이터/통계) | 합성 스타일 (데이터 강화) | 구조화 데이터가 핵심 |
| CAT6 (AI 안전/윤리) | 박종훈 + 합성 혼합 | 맥락 + 실용 가이드 필요 |
